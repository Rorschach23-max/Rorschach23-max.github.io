对于Git一些操作 通过各种区来理解

Workspace：工作区

本地电脑上的东西都在这个区

从远程仓库克隆到本地

从远程仓库克隆特定分支到本地

远程仓库有多个分支时，切换到想要更新的分支

（获取fetch 然后 合并merge ） 或者 pull抓取 等同这两个操作

Index / Stage：暂存区

要提交 首先得把工作区的更改放到暂存区

提交某个文件

**提交全部文件 **

Repository：仓库区（或本地仓库）

要把暂存区的变更提交到仓库

Remote：远程仓库

推送到master分支

推送本地分支到远程仓库

设置推送的新分支为上游分支，之后每次提交只需用 git push

合并dev分支到master分支 然后上传
